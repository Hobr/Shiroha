# 设计原则

- **KISS**: 每个 crate 只负责一个关注点，避免一揽子模块。
- **正交解耦**: 状态机语义、WASM 执行、分发、聚合、传输相互独立，可单独替换或抽换实现。
- **主控有状态, 节点无状态**: 状态迁移、事件日志、FSM 实例均驻留主控；节点仅按请求执行 Action。
- **主控可兼职节点**: 通过本地 Transport 实现共址执行，不引入特殊路径。
- **接口先于实现**: gRPC 为首发传输，Quic 与消息队列在 trait 层预留，不改上游调用方。
- **用户前端可替换**: 主控仅暴露 gRPC 服务；CLI 为首发客户端，未来 TUI / Web 复用同一服务。
- **声明式分发**: 分发与聚合策略在 FSM 定义中以元数据形式声明，主控解释执行，避免散落在业务代码中。
