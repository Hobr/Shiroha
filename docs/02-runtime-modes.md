# 运行模式

> 文档定位：本文描述目标运行模式和职责边界，不代表三种模式都已在当前仓库中完成实现。

单一二进制 `shirohad`，通过配置文件决定运行模式：

| 模式 | 说明 |
|------|------|
| **controller** | 管理状态机生命周期，驱动状态流转，分发任务，聚合结果，对外暴露管理接口 |
| **node** | 无状态执行节点，接收任务，执行 WASM Action，返回结果 |
| **standalone** | Controller + Node 合体，本地执行所有任务，适用于开发/单机部署 |

Controller 模式自身也可同时充当 Node（配置开关），省去单独部署节点的开销。
