# 契约与 ABI

远期 [roadmap](12-roadmap.md) 将开放由 WASM 实现的自定义分发 / 聚合策略。一旦这些数据形状跨越 WIT 边界，即成为**冻结的 ABI**，不能再做破坏性变更。本章列出需要提前钉死的最小字段集与演进规则。

## 冻结范围

以下对象的公开字段一经发布进入稳定期：

- **ActionMeta**: Action 名、分发声明、聚合声明、输入/输出的逻辑形状标签。
- **DispatchPlan**: 目标节点标识集合、扇出方式、分片编号、每节点超时。
- **NodeView**: 节点标识、标签、能力、健康状态、当前在途任务数。分发策略会基于此视图决策。
- **ActionResult**: 结果状态 (成功/失败/超时)、`attempt_id`、节点标识、载荷 (bytes)、诊断信息。
- **事件记录**: `event_id`、`event_seq`、实例标识、事件类型、提交时间戳、触发者。

## 演进规则

- **只增不减**: 新增字段时为可选，旧 guest 可忽略。
- **枚举扩展**: 枚举变体允许追加，guest 必须有 `unknown` 兜底分支。
- **语义冻结**: 已发布字段的含义不得变更，即便类型兼容。
- **版本号**: 每个冻结对象有独立的 schema 版本；破坏性变更升主版本并保留旧版本过渡期。
- **WIT 兼容**: WIT 世界遵循半版本号规则；破坏性变更要求 FSM 重新部署。

## 未冻结但建议稳定的部分

- `shiroha-dispatch` / `shiroha-aggregate` 的 Rust trait 签名。只要不开放给 WASM，Rust 侧破坏性变更仅影响一方，但高频变更会拖慢实现节奏。
- gRPC 的服务方法列表，详见 [grpc](08-grpc.md) 的协议演进小节。

## 落地时机

- M0–M2 期间只对 Rust 侧 trait 负责；WIT 中暂不暴露上述 ABI，保留迭代空间。
- 在着手「WASM 自定义策略」(M4 之后) 前，须先发布一版 `wit/shiroha-strategy.wit` 并通过契约评审，之后进入冻结期。
