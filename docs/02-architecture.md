# 分层架构

```
┌─────────────────────────────────────────────────────────┐
│  前端交互: CLI (现在) / TUI / Web (未来)                 │  apps/sctl        
├─────────────────────────────────────────────────────────┤
│  主控端 Controller (有状态)                              │  apps/shirohad    
│    ├─ FSM 实例管理 / 调度 / 聚合入口                     │
│    └─ 暴露 gRPC: OrchestratorSvc + NodeSvc               │
├─────────────────────────────────────────────────────────┤
│  节点端 Agent (无状态)                                   │  apps/shiroha-agent
│    └─ 执行 WASM Action, 可同机共址为主控的本地执行器     │
└─────────────────────────────────────────────────────────┘
          │                 │                  │
  ┌───────┴──────┐ ┌────────┴───────┐ ┌────────┴─────────┐
  │ shiroha-core │ │ shiroha-wasm   │ │ shiroha-transport│
  │  (纯 FSM)    │ │ (Wasmtime+WIT) │ │  (trait+注册表)  │
  └──────────────┘ └────────────────┘ └──────────────────┘
          │                 │                  │
  ┌───────┴──────┐ ┌────────┴───────┐ ┌────────┴──────────┐
  │ shiroha-     │ │ shiroha-wit    │ │ shiroha-transport-│
  │ dispatch +   │ │ (WIT 接口)     │ │ grpc (+quic/mq 预 │
  │ aggregate    │ │                │ │ 留)               │
  └──────────────┘ └────────────────┘ └───────────────────┘
          │
  ┌───────┴────────┐
  │ shiroha-storage│ (redb, 仅主控)
  └────────────────┘
```

## 角色

- **主控端 (Controller)**: 保管 FSM 定义、实例、事件日志；调度 Action 分发；聚合结果；对外暴露两个 gRPC 服务。
- **节点端 (Agent)**: 纯执行单元，按 Action 请求运行 WASM，不保留业务状态。
- **客户端 (Client)**: 通过主控的 `OrchestratorService` 进行部署、触发、查询。CLI 是首发形态。

## 共址模式

主控进程可内嵌一个本地 Agent，通过 Transport 的本地实现直接调用，与远程节点在上层接口上等价。单机场景不需要任何额外部署。
