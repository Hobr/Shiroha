# 事件溯源

每次状态转移记录为不可变事件日志，天然适合状态机场景。

## 事件记录

每条事件包含：时间戳、Job ID、事件类型（Created / Event / ActionComplete / Completed 等）、相关元数据（flow 名称、版本、状态转移、action 名称、执行 Node、执行结果等）。

## 作用

- **审计追踪**：完整记录谁在什么时候触发了什么操作
- **故障恢复**：Controller 重启后从事件日志重建 Job 状态
- **调试**：回放事件序列定位问题
- **分析**：统计状态停留时间、转移频率、失败分布

## 实现

- 事件日志写入 Storage（与 Job 状态同一后端）
- 写入和状态更新在同一事务内，保证一致性
- Phase 1 写入即可，查询/回放 API 后续迭代
