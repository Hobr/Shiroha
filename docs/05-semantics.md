# 语义模型

本章规定框架在正确性维度的不变量。任何实现都必须满足这里的约束，否则状态机语义将被破坏。

## 事件串行化

- 同一 FSM 实例的事件必须**严格串行**处理。主控对每个实例维护一条有序队列。
- 并发 `SubmitEvent` 进入队列后按到达顺序执行；不同实例之间互不阻塞。
- `SubmitEvent` 的 ack 语义需在接口文档中明确选择其一：
  - **Accepted**: 事件已排入队列即返回 (异步完成)。
  - **Applied**: 事件已完成状态迁移并落库后返回 (同步完成)。
- 本项目默认采用 **Applied** 语义，并另行提供显式的 fire-and-forget 变体。

## 交付语义

- 节点执行 Action 默认为 **at-least-once**。重试由调度器在节点超时 / 结果丢失时发起。
- Action 的副作用幂等性由 WASM 作者保证，框架提供 `attempt_id` 作为稳定去重键：`(instance_id, event_seq, action_name, shard_index, retry_index)`。
- `attempt_id` 通过 WIT 的 action 上下文传入 guest；guest 可据此查询自有去重表。

## 超时与故障判定

- **调用超时 (invocation timeout)**: 主控对每次 Action 调用设置的上限；到期触发再调度。
- **心跳超时 (liveness timeout)**: 节点长时间未心跳则标记下线，其在途任务全部重排。
- 两档超时独立配置；调用超时 < 心跳超时。
- 结果丢失与执行失败不可区分时，一律按 at-least-once 重发，由幂等性兜底。
- 节点在接受任务时先 ack，确保「任务未到达」可被主控区分于「已执行但结果丢失」。

## 定时器与系统事件

- 某些转换由「状态停留时长」触发 (例如：进入 `Waiting` 后 30s 未收到事件则转 `Timeout`)。
- 框架内置持久化的**超时调度器**：在状态进入时登记到期时刻，到期等价于主控向实例队列注入一条系统事件。
- 超时记录与 FSM 实例状态在同一事务中写入，避免「状态已切换、超时未登记」的漏登。
- 重启后由 storage 恢复未到期的超时。

## 聚合失败的处置

- `AllOk`: 任一节点失败，本次 Action 结果为失败；状态迁移不发生，事件计入日志但标注 `aborted`。
- `Quorum(n)`: 达不到法定数则失败，同上。
- `First`: 首个成功返回即结束；其余节点的结果被丢弃 (仍可能已产生副作用，依赖幂等性)。
- `Reduce`: 由 WASM 决定是否失败。

聚合失败不触发自动补偿；补偿由用户在状态机中显式建模 (迁移到错误态并在错误态的 on-enter 中做回滚)。

## 序列化契约

- **框架消息** (FsmDef、事件、DispatchPlan、ActionResult 头部): 选用 `rkyv` 作为规范格式，保证零拷贝与版本兼容。
- **Action payload** (input/output): WIT 层面为 `list<u8>`；主控不解释内容，由 FSM 定义的作者在 guest 侧自行约定编码。
- 框架消息字段演进遵循 [contracts](09-contracts.md) 的兼容性规则。
